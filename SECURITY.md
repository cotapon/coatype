# Security Policy

CoAType は **マイク音声・API キー・アクセシビリティ権限** を扱うアプリのため、
セキュリティ上の問題には真剣に対応します。

## 脆弱性の報告方法

**公開 Issue を立てないでください。** 脆弱性の詳細を公開すると、修正前に悪用される恐れがあります。

代わりに、以下のいずれかで非公開に報告してください:

1. **GitHub Private Vulnerability Reporting（推奨）**
   このリポジトリの **Security タブ → "Report a vulnerability"** から報告してください。
   （メンテナが GitHub の Private vulnerability reporting を有効化している必要があります。）

2. **メール**
   上記が使えない場合は、リポジトリオーナー（[@cotapon](https://github.com/cotapon)）宛に連絡してください。

報告には以下を含めていただけると助かります:

- 影響を受けるバージョン
- 再現手順または PoC
- 想定される影響（情報漏洩・権限昇格など）

## 対応の流れ

- 受領後、できるだけ早く（目安: 数日以内）一次返信します。
- 修正方針を相談し、パッチをリリースします。
- 報告者のクレジットを希望される場合は、リリースノートに記載します。

## 対象範囲の補足

- API キーは `COATYPE_API_KEY` 環境変数または macOS Keychain にのみ保存されます。リポジトリにキーをハードコードする実装は存在しません。
- ビルド成果物の署名鍵・証明書は GitHub Actions の Secrets で管理され、ソースには含まれません。
